SejongMate

メインポスター

※すべての成果物は個人ではなく、素晴らしいチームメンバーと共同で作業しました。

このプロジェクトは、大規模言語モデルを活用した世宗大学のチャットボットです。 2023年の学内ソフトウェア創造設計コンテストで2位を獲得しました。 以下に、私の主な役割を詳しく説明します。

私の仕事

1. データの収集、クレンジング、および拡張

主な業務
  • 多様なソース（大学コミュニティ、学事内規、ウィキペディア）からデータを収集し、前処理するプロセスを主導しました。
  • クローリング、データクレンジング、拡張などのプロセスを含むデータパイプラインを設計し実装しました。
  • データ拡張、ランダム文順変換、ランダム文削除などの技法を適用し、データの多様性と品質を改善しました。
  • QAペア形式の構造化データを生成し、DBを通じて管理しました。

    • 主な課題と解決策の議論

    課題1. 広範囲のクローリングによる核心キーワード選定の困難さ

    初期データ収集過程でユーザーの需要を把握し、 それに合うキーワードを基にデータを収集する必要性を感じました。 大学コミュニティの最近の投稿から疑問符で終わる質問を抽出して調査し、 さらにアンケート調査を通じてキーワードを見出しました。
    見出したキーワードと内部会議を通じて必須的に含めるべきキーワードを最終決定し、 これを通じて大学コミュニティ、学事内規から情報を抽出してデータを構築、拡張しました。

    課題2. Question-Answerペアデータの生成

    内部会議を通じてチャットボットの核心機能をユーザークエリと私たちのDBの最も類似した質問との類似度を計算し、 それに適した回答を出力することに決めました。 これに従ってQuestion-Answerデータを生成する必要があり、プロセスは次の通りでした。

    1. 前述の収集した文書をGPT inputに適したチャンク単位に分割
    2. GPT-turbo-3.5 apiを活用して前述の収集された文書に対するQuestion生成
    3. 生成されたQuestion + 文書を入力として信頼性のあるAnswerを生成
    4. 人間による最終検証
    各プロセスには適切なGPTプロンプトを直接作成して効率を高めました。

    課題3. 収集されたデータの限られた多様性

    理論と異なり、実際のユーザーは期待していた文章とは異なる方法で質問をすることが多いです。 例えば、大学名を略したり、科目名を略したりする方法があります。 私たちはこれを考慮して、任意に類義語/略語に関する質問データを拡張しました。
    また、韓国語は語順が変わっても意味が保たれる場合が多いです。 私たちはこれを考慮して、文章内の単語をランダムに入れ替えてデータを拡張しました。
    最終的に拡張されたデータは収集したQAに対して約4万件です。

    1. GPTプロンプトを通じて質問フォーマットを指定し、多様な質問タイプを生成
    2. 類義語/略語語彙に関する拡張
    3. ランダム文順変換および削除による追加データ多様化

    2. モデルのデプロイ

    主な業務
  • AWS EC2を活用したAIモデルのデプロイ。
  • FlaskベースのAPI開発によるモデルデプロイとデータパイプラインの構築。
  • MySQLを使用したリレーショナルデータベースの設計と運用。

    • 主な課題と解決策の議論

    課題1. AWS EC2インスタンスの選定

    EC2インスタンスのコストに適したモデルを選定するプロセスは難しいものでした。私たちに割り当てられた予算は 限られており、使用したいモデルは巨大でした。 私たちはできるだけ軽量なモデルを使用し、量子化（LoRA）を通じてモデルサイズを縮小し、 EC2インスタンスタイプとしてt2.mediumを使用して最大限メモリを増やそうとしました。

    課題2. Flaskを通じたAPIデプロイ

    LLMをデプロイしバックエンドと連携する作業は、当時情報が少なく難しい作業でした。 デプロイ過程での主な考慮事項はモデルの推論時間と出力の視覚化方式でした。 私たちは推論時間を 私たちは前述のサーバーにアップロードしたモデルをFlaskを通じてAPIとして提供し、Gradioを通じてモデル出力をストリーミング方式で視覚化してクライアントに提供しました。 主なプロセスは次の通りです：

    1. AWSでEC2インスタンスを作成し、必要な環境を設定
    2. モデルをEC2インスタンスにアップロード
    3. Flaskを利用してモデルをAPIとして提供
    4. ユーザーインタラクションのためのGradioインターフェースの活用
    5. GradioインターフェースとFlask APIを接続し、システムテスト

    ForeignAICC

    メインポスター

    ※すべての成果物は個人ではなく、素晴らしいチームメンバーと共同で作業しました。

    本プロジェクトは、ソウル市が主催する社会問題解決のための生成型AI競技大会に参加しています。 私たちのプロジェクトは、増加する外国人労働者に対して減少する外国人労働者支援センターを社会問題として取り上げており、現在、予選を全体4位で通過し、本選に進出した状態です。 (8月2日本選大会)

    主な特徴

    1. 外国人労働者が知りたいこと（法律、ビザの質問など）についてチャットボットに相談します。
    2. チャットボットはデータに基づいて、法的情報、例、具体的な解決策を提供します。
    3. さらなるカウンセリングが必要な場合、アプリ内から直接相談のスケジュールを組むことができます。
    4. オフライン相談の労力を減らすため、カウンセラーにチャットの要約を提供し、プロセスをよりスムーズにするのに役立てます。
